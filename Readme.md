# Manual Testing - Test Cases, Scenarios, Mindmaps, Reports, and Test Metrics

This repository contains comprehensive test artifacts for the **Manual Testing** of the **Secure Bank** project, a banking web application. The focus is on critical modules such as **Sign Up**, **Sign In**, **Forget Password**, and **Transactions**. These artifacts include mindmaps, test cases, scenarios, execution reports, bug reports, and test metrics to streamline the testing process and ensure the application's quality.

## Introduction

Manual testing is crucial to ensure the quality and reliability of the **Secure Bank** web application. This repository serves as a centralized hub for organizing and storing test artifacts. These resources are designed to assist in effectively planning, executing, and tracking the testing activities for the project.

---

## Project Overview

The **Secure Bank** project is a banking web application developed to provide seamless money transaction services in **Bangladesh**. Key features include:
- **Sign Up:** Users can create an account.
- **Sign In:** Users can log into their accounts securely.
- **Forget Password:** Password recovery functionality.
- **Transactions:** Smooth and secure money transfers.

## Test Artifacts

The repository includes the following test artifacts:
1. **Mindmaps**  
   Visual representations of the testing process, showing test coverage and relationships between test components.

2. **Test Case Scenarios**  
   High-level test scenarios to identify testing objectives and scope.

3. **Test Cases**  
   Detailed steps for each test scenario, including test case ID, prerequisites, steps, and expected results.

4. **Test Metrics**  
   Comprehensive reports to measure and track the effectiveness of the testing process.

5. **Bug Reports**  
   Documentation of identified issues with detailed descriptions, severity levels, and suggested resolutions.

## FAQ

### Q1: How can I contribute to this project?
**A:** Contributions are encouraged! Add new test artifacts like test cases, scenarios, mindmaps, or reports, and submit a pull request with your updates.

### Q2: Can I use the test cases and scenarios for my own projects?  
**A:** Yes! These artifacts are intended for educational and practical use. Feel free to adapt them to your projects.

### Q3: Is there a specific format for test cases?  
**A:** We recommend including the following fields:
- Test Case ID
- Description
- Pre-requisites
- Test Steps
- Expected Results  

Consistency across test cases ensures readability and usability.

### Q4: Are all test processes integrated into one file?  
**A:** Yes, a consolidated Excel file includes test cases, scenarios, mindmaps, reports, and metrics for simplicity.
